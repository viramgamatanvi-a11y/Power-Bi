4. What is a POWER BI WORKFLOW?
=> Connect → Transform → Load → Model → DAX → Visualize → Publish
=> Report View -> table View -> Model View -> DAX Query View -> TMDL View
=> Power BI Workflow me 2 Types hote hai
1. Power BI “Back-End” => Power Query Editor
2. Power BI “Front-End” => Model View , Data View , Report View
=> Power Query Editor = Power Query Editor is a tool in Power BI used to clean, transform, and prepare data before loading it into the model.
=> Model View = Relationship bana sakte hai two table ke beach me.
=> Data View (Table View) = Data ko table format me dekhna , New columns (calculated columns) banana , Column ka data type and format change kr sakte hai.
=> Report View = Isme hum charts, tables, aur dashboards bana sakte hai.
=> ETL(Extract, Transform, Load)
=>• The front-end includes the Data, Model \& Report views, where most of the modeling, analysis and visualization takes place
• The back-end includes the Power Query Editor, where raw data is extracted, transformed, and loaded to the front-end (ETL)


5. What is a different between BACK-END and FRONT-END?
=> BACK-END
-Connect \& extract data
→ Different sources se data import karna (Excel, SQL, etc.)
-Profile \& QA data

=> FRONT-END
-Build data models (relationships)
→ Tables ke beech connection banana (primary key \& foreign key use karke)

→ Formulas use karke calculations banana (SUM, COUNT, etc.)
-Design reports & dashboards
→ Data ko charts aur graphs me convert karna (interactive report banana)
-Publish & share (Power BI Service)
→ Report ko cloud par upload karke dusro ke saath share karna 👍

	Product		Sales

	Laptop		700

=> Append Karne se Rows Add hoti hai, Column nahi.


NOTE =========== Merge = Column Add (Side me)
		 Append = Rows Add (Niche)

59. What is a RELATED?

👉 To RELATED function se tum dusri table ka column apni table me la sakti ho

🔹 Syntax

RELATED(ColumnName)

🔹 Example
Sales table me ProductID hai
Product table me Product Name hai

👉 Formula:

🔹 Important Points
Relationship hona zaruri hai
Ye row-wise kaam karta hai (row context)
Mostly calculated column me use hota hai
Direct measure me use nahi hota (unless inside SUMX, FILTER, etc.)


60. What is a CALCULATE?
=>CALCULATE() ka use hota hai kisi calculation ko naye filters ke saath evaluate karne ke liye
👉 Ye existing filters ko change ya override kar sakta hai

👉 Tum ek calculation likhti ho
👉 Aur us par apne filters apply karti ho
👉 Result us naye filter context ke hisaab se aata hai

🔹 Syntax
CALCULATE(Expression, Filter1, Filter2, ...)

🔹 Example
USA Sales =
CALCULATE(
    [Total Sales],
    'Territory Lookup'[Country] = "USA"
)

👉 Ye sirf USA ka sales calculate karega (chahe report me kuch bhi filter ho)

🔹 Important Points
Filters add ya replace kar sakte ho
Measure ya formula use hota hai expression me
Filters me column-based condition hoti hai (measure nahi)

CALCULATE apna filter force karta hai
Jo pehle se filter hai usko replace kar deta hai

64. What is a ITERATOR FUNCTIONS?
=> Iterator functions wo functions hote hain jo table ki har row par loop (iterate) karte hain
👉 Aur har row par calculation karke final result aggregate (SUM, AVG, etc.) karte hain

👉 Step-by-step:

Table lo
Har row par formula apply karo
Fir sab results ko combine (SUM/AVG) karo
🔹 Example (SUMX)
SUMX(Sales, Sales[Price] * Sales[Quantity])

👉 Har row ke liye:

Price × Quantity calculate hoga
👉 Fir sabka total sum milega

🔹 Important Points
Row-by-row calculation hota hai
Fir aggregation hota hai (SUM, COUNT, AVG, etc.)
Temporary column jaisa behave karta hai

🔹 Common Iterator Functions
SUMX → Sum of calculated values
COUNTX → Count
AVERAGEX → Average
MAXX / MINX → Max / Min
RANKX → Ranking
🔹 Hinglish

Iterator ka matlab:
👉 Har row par calculation karo
👉 Fir sabko combine karo

🔹 Short Line

👉 Iterator (X) = Row-wise calculation + final aggregation 👍


65. What is a TIME INTELLIGENCE?
=> Time Intelligence ka use hota hai date-based calculations karne ke liye
👉 Jaise: last month, year-to-date, running total, etc.

🔹 1. Running Total (Rolling Period)
CALCULATE(
    [Measure],
    DATESINPERIOD(Calendar[Date], MAX(Calendar[Date]), -10, DAY)
)

👉 Last 10 days ka data calculate karta hai
👉 Dynamic hota hai (date ke hisaab se change hota hai)

🔹 2. Performance To-Date (YTD, MTD, QTD)
CALCULATE([Measure], DATESYTD(Calendar[Date]))

👉 Year start se ab tak ka total (Year-To-Date)

👉 Similar:

DATESMTD → Month-to-date
DATESQTD → Quarter-to-date

🔹 3. Previous Period
CALCULATE(
    [Measure],
    DATEADD(Calendar[Date], -1, MONTH)
)

👉 Previous month ka data return karta hai

🔹 Hinglish Samajh
Running Total → Last kuch days/months ka total
Performance To Date → Starting se ab tak ka total
Previous Period → Pichla month/year compare
🔹 Pro Tip

👉 Moving average =
Running total ÷ number of days/months

🔹 Short Line

👉 Time Intelligence = Date ke hisaab se smart calculations 👍

Minimize iterator functions
