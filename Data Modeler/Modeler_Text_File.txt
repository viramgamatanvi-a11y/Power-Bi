   3. Demonstrate Cardinality

     1 : Many(One-to-Many)
     In a one-to-many relationship, a key value appears once in one table but can appear multiple times in another table.
     Example: Customer_ Dim → Sales_ Fact

     Many : 1(Many-to-One)
     In a one-to-many relationship, a key value appears once in one table but can appear multiple times in another table.
     Example: Sales_ Fact → Product_ Dim

     1 : 1(One-to-One)
     In a one-to-one relationship, a key value appears only once in both tables.    
     Example: Sales_ Fact → Returns_ Fact 

=> 3. Advanced Model Settings 

      1. Set Relationship Cardinality and Cross Filter
        
        -Relationships are set to One-to-Many.
        -Cross filter direction is kept Single for better performance.

      2. Bidirectional Filters

        -Bidirectional filters are disabled by default.
        -They are enabled only when needed.

      3. Inactive Relationship Simulation

        -An inactive relationship is created between Date_ Dim and Returns_ Fact.
        -It is used in measures with the USERELATIONSHIP() function.

      4. Resolve Filter Ambiguity

        -Relationships are checked in the Model view.
        -Single filter direction is used to avoid circular filtering.
