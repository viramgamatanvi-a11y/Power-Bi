3\. What is a different between Excel and power BI?

=> | Scenario      | Excel                          | Power BI                               |

| ------------- | ------------------------------------ | -------------------------------------------- |

| Data size     | Small (few thousand rows)            | Large (lakhs / millions rows)                |

| Purpose       | Data entry, formulas, quick analysis | Dashboards, reporting, insights              |

| Visualization | Basic charts                         | Interactive dashboards (filters, drill-down) |

| Sharing       | File share (.xlsx)                   | Online sharing (live dashboards)             |

| Automation    | Manual refresh                       | Auto refresh (real-time possible)            |

| Data sources  | Limited                              | Multiple (SQL, APIs, cloud, etc.)            |



4\. What is a POWER BI WORKFLOW?

=> Connect → Transform → Load → Model → DAX → Visualize → Publish

=> Report View -> table View -> Model View -> DAX Query View -> TMDL View

=> Power BI Workflow me 2 Types hote hai

&#x20;  1. Power BI “Back-End” => Power Query Editor

&#x20;  2. Power BI “Front-End” => Model View , Data View , Report View

=> Power Query Editor = Power Query Editor is a tool in Power BI used to clean, transform, and prepare data before loading it into the model.

=> Model View = Relationship bana sakte hai two table ke beach me.

=> Data View (Table View) = Data ko table format me dekhna , New columns (calculated columns) banana , Column ka data type and format change kr sakte hai.

=> Report View = Isme hum charts, tables, aur dashboards bana sakte hai.

=> ETL(Extract, Transform, Load)

=>• The front-end includes the Data, Model \& Report views, where most of the modeling, analysis and visualization takes place

• The back-end includes the Power Query Editor, where raw data is extracted, transformed, and loaded to the front-end (ETL)



5\. What is a different between BACK-END and FRONT-END?

=> BACK-END

\-Connect \& extract data

→ Different sources se data import karna (Excel, SQL, etc.)

\-Profile \& QA data

→ Data ko check aur clean karna (errors, null values remove)

\-Transform \& shape tables

→ Data ko modify karna (filter, sort, new columns banana, grouping)

\-Merge / Append queries

→ Multiple tables ko join ya combine karna

\-Advanced transformation (M code)

→ Complex changes ke liye advanced coding use karna



=> FRONT-END

\-Build data models (relationships)

→ Tables ke beech connection banana (primary key \& foreign key use karke)

\-Add calculated measures \& columns (DAX)

→ Formulas use karke calculations banana (SUM, COUNT, etc.)

\-Design reports \& dashboards

→ Data ko charts aur graphs me convert karna (interactive report banana)

\-Publish \& share (Power BI Service)

→ Report ko cloud par upload karke dusro ke saath share karna 👍

8\. What is a QUERY EDITING TOOLS?

=> The HOME tab includes general settings and common table transformation tools.

=> The TRANSFORM tab includes tools to modify existing columns (splitting/grouping, transposing, extracting text, etc.)

=> The ADD COLUMN tools create new columns (based on conditional rules, text operations, calculations, dates, etc.)



9\. What is STORAGE \& CONNECTION MODES?

=> 1. Import

&#x20;  Data Power BI ke andar store ho jata hai (memory me)

&#x20;  Reports fast chalte hain (default mode)

=> 2. Direct Query

&#x20;  Data Power BI me store nahi hota

&#x20;  Direct database se live query run hoti hai

&#x20;  Real-time data milta hai

=> 3. Composite Model (Dual)

&#x20;  Kuch data import hota hai aur kuch DirectQuery se aata hai

&#x20;  Dono ka mix use hota hai

=> 4. Live Connection

&#x20;  Already published dataset (Power BI Service / Azure) se direct connect karte ho

&#x20;  Data ko change nahi kar sakte, sirf use kar sakte ho



10\. What is a DATA PROFILING: COLUMN QUALITY?

=> Profiling tools like column quality, column distribution, and column profile

=> 1. column quality = Column quality batata hai ki column ka data kitna correct hai.(Valid data, Error value, Empty value).

=> 2. column distribution = column distribution batata hai ki column me value ka distribution kaisa hai (Unique value, Duplicate Value).

=> 3. column profile = column profile column ki detailed information deta hai (Min ,Max , Average ,value count).
